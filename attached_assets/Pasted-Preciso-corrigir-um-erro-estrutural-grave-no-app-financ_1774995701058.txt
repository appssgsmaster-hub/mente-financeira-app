Preciso corrigir um erro estrutural grave no app financeiro.

CENÁRIO ATUAL:
- Não existe nenhuma transação no histórico
- Não existe nenhuma entrada
- Não existe nenhuma saída
- Mesmo assim o dashboard continua mostrando saldo no “Ecossistema Total”
- E as contas continuam com valores distribuídos

Exemplo atual:
- Ecossistema Total ainda mostra €3.919,51
- As contas continuam com saldo:
  - Reserva: €117,58
  - Pro-labore: €391,95
  - Tax: €78,40
  - Lucro: €783,90
  - Operacional: €1.371,83
  - COGS: €1.175,85

Isso é saldo fantasma.
Não há transações, então todos esses valores deveriam ser ZERO.

OBJETIVO:
Eliminar completamente qualquer saldo fantasma e garantir que, se não houver transações, o ecossistema e todas as contas mostrem €0,00.

DIAGNÓSTICO PROVÁVEL:
O sistema provavelmente está com um ou mais destes problemas:
1. saldo persistido diretamente nas contas
2. saldo do ecossistema salvo separadamente
3. deleção apaga apenas o histórico visível, mas não apaga os dados-base
4. cache/localStorage/sessionStorage mantendo valores antigos
5. estado global React/Zustand/Context não sendo resetado corretamente
6. banco de dados com registros órfãos
7. cálculo do dashboard lendo de uma fonte diferente do histórico de transações

REGRAS CORRETAS:
1. Se não houver nenhuma transação, então:
   - Ecossistema Total = 0
   - Entradas = 0
   - Saídas = 0
   - Todas as contas = 0

2. O saldo das contas NÃO pode existir sozinho como valor independente permanente, a menos que seja reconstruído a partir de transações reais.
3. O Ecossistema Total deve ser sempre derivado de dados reais, nunca de um valor residual salvo separadamente.
4. Ao apagar todas as transações, o sistema deve recalcular tudo do zero e zerar completamente o dashboard.
5. O histórico, as contas e o ecossistema devem usar a MESMA fonte de verdade.

O QUE PRECISO QUE VOCÊ VERIFIQUE:
1. Todos os lugares onde saldo pode estar sendo persistido:
   - banco de dados
   - localStorage
   - sessionStorage
   - Zustand/store/context
   - cache de componentes
   - tabelas auxiliares de balances/accounts/summary

2. Verifique se existe:
   - tabela de transactions
   - tabela de account balances
   - tabela de summary/ecosystem
   - campos redundantes de saldo salvos no usuário/empresa

3. Verifique se o delete:
   - remove só a transação visual
   - ou realmente dispara recálculo total das contas e do ecossistema

4. Verifique se a UI está lendo de uma fonte e o histórico de outra.

CORREÇÃO NECESSÁRIA:
1. Implementar uma função única de reconciliação total do sistema financeiro.
2. Essa função deve:
   - ler todas as transações reais
   - recalcular todas as contas do zero
   - recalcular o ecossistema total do zero
   - atualizar a UI com os valores corretos
3. Se a lista de transações estiver vazia:
   - zerar imediatamente todas as contas
   - zerar imediatamente o ecossistema
   - zerar total de entradas
   - zerar total de saídas

4. Criar um botão/função interna de “reset financeiro seguro” para ambiente de desenvolvimento, que:
   - limpe transações
   - limpe saldos persistidos
   - limpe cache/localStorage relacionado
   - force recálculo completo

5. Garantir que o sistema nunca dependa de saldo antigo salvo sem correspondência com transações.
