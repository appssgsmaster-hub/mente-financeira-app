Preciso corrigir um erro crítico de consistência financeira no app.

PROBLEMA
As entradas e saídas registradas no sistema NÃO estão refletindo corretamente na divisão das contas.
Além disso:
1. O valor do “Ecossistema Total” não bate com a soma dos saldos das contas.
2. Quando registro uma entrada ou saída, o sistema às vezes atualiza o total geral, mas não redistribui corretamente entre as contas.
3. Em alguns casos, a conta operacional fica negativa sem o restante do sistema acompanhar corretamente.
4. O sistema precisa recalcular tudo de forma consistente e automática sempre que houver qualquer alteração financeira.

O QUE PRECISA ACONTECER
Sempre que o usuário:
- registrar uma entrada
- registrar uma saída
- editar uma entrada
- editar uma saída
- excluir uma entrada
- excluir uma saída
- alterar percentuais das contas
- marcar um compromisso como pago
- editar compromisso
- excluir compromisso
- recalcular saldos manualmente

o sistema deve recalcular de forma global e consistente:

1. saldo total do ecossistema
2. saldo individual de cada conta
3. distribuição ideal por percentual
4. alertas do ecossistema
5. compromissos/pagamentos relacionados
6. cards e dashboards que dependem desses dados

REGRA DE NEGÓCIO CORRETA
O app precisa ter uma única fonte de verdade para os cálculos.

Regra esperada:
- Ecossistema Total = Total de Entradas - Total de Saídas
- Soma de todas as contas = Ecossistema Total
- Nunca pode haver diferença entre:
  a) valor do ecossistema
  b) soma dos saldos das contas

As contas não podem ficar “soltas” com valores antigos em cache.

COMPORTAMENTO ESPERADO DAS ENTRADAS
Quando lançar uma ENTRADA:
- a entrada deve aumentar o total do ecossistema
- a entrada deve ser distribuída entre as contas conforme o percentual atual de cada conta
- a soma da distribuição precisa bater exatamente com o valor da entrada
- se houver problema de arredondamento, ajustar centavos na última conta para fechar 100%

COMPORTAMENTO ESPERADO DAS SAÍDAS
Quando lançar uma SAÍDA:
- a saída deve reduzir o ecossistema total
- a saída deve sair da conta correspondente escolhida pelo usuário
- se a saída for de compromisso planejado, deve atualizar o compromisso e o saldo da conta correta
- se a saída afetar uma conta específica, isso precisa refletir também no dashboard geral

IMPORTANTE
Hoje parece que existem cálculos duplicados ou estados separados.
Quero que você revise isso e centralize a lógica de cálculo em funções únicas, evitando que:
- um card mostre um valor
- outro card mostre outro
- e o total geral mostre outro diferente

IMPLEMENTAÇÃO TÉCNICA
Quero que você:

1. Identifique onde os cálculos estão sendo feitos hoje
   - dashboard
   - contas
   - entradas/saídas
   - compromissos
   - botão “Recalcular Saldos”

2. Crie uma função central única, algo como:
   - recalculateFinancialState()
ou nome equivalente,
responsável por recalcular:
   - totalEntradas
   - totalSaidas
   - ecosystemTotal
   - accountBalances
   - projected commitments
   - alerts

3. Garanta que TODOS os componentes usem essa função/fonte única de dados

4. Elimine qualquer cálculo local duplicado dentro dos componentes

5. Garanta atualização reativa imediata após qualquer alteração

6. Se houver uso de localStorage, banco, cache ou estado global, garantir sincronização correta após salvar/editar/excluir

7. Revisar arredondamento para evitar diferenças de centavos entre soma das contas e total do ecossistema

Crie e valide estes cenários:

Cenário 1
- entrada de €1000
- percentuais das contas somando 100%
Resultado:
- ecossistema total = €1000
- soma das contas = €1000

Cenário 2

Resultado:
- ecossistema total = €800
- conta operacional reduzida corretamente
- soma das contas = €800

Cenário 3

Resultado:
- ecossistema total e contas recalculados corretamente

Cenário 4

Resultado:
- valores recompostos corretamente sem resíduos antigos

Cenário 5
- alterar percentuais das contas
Resultado:
- redistribuição correta do saldo conforme nova configuração
- total continua batendo

Cenário 6

Resultado:
- reduz saldo da conta correta
- atualiza alertas

- ecossistema continua consistente

Antes de finalizar, quero uma checagem automática:
- comparar soma das contas com o ecossistema total
- se houver diferença, registrar erro de debug no console
- idealmente bloquear inconsistência silenciosa

- total do ecossistema
- soma das contas
- diferença entre soma das contas e ecossistema
